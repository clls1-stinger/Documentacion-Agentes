# INCIDENT REPORT: 2026-02-04 - n8n Database Crash

## Resumen
El servicio n8n entró en un "Crash Loop" tras una aparente actualización fallida. No podía arrancar debido a una discrepancia en el esquema de la base de datos.

## Detalles Técnicos
- **Error**: `SQLITE_ERROR: no such column: User.role`
- **Versión Afectada**: n8n v2.6.3
- **Archivo**: `/home/emky/.n8n/database.sqlite`

## Solución Aplicada
Se realizó una intervención manual (Hot Patch) en la base de datos SQLite.

1. **Backup**: Se creó `database.sqlite.bak`.
2. **Patch**:
   ```sql
   ALTER TABLE "User" ADD COLUMN "role" varchar DEFAULT 'global:member';
   ```
3. **Reinicio**: `pm2 restart n8n`.

## Resultado
Servicio restaurado. El esquema ahora es compatible con la versión v2.6.3.
