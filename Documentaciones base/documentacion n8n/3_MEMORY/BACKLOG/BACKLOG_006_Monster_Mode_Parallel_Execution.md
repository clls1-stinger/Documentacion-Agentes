# 👹 BACKLOG 006: "Monster Mode" - Massive Parallel Map-Reduce

> "I want the thing to get monstrous." - User

## Objective
Implement a high-scale parallel execution engine capable of handling **thousands of simultaneous actions** (e.g., spawning 1000 terminal instances) generated by the Gemini Actor, followed by an intelligent aggregation and summarization phase.

## Architecture: Map-Reduce for AI Agents

### 1. Map Phase (The Monster) 🐉
- **Trigger**: Gemini Actor outputs a massive array of actions (N > 10, up to 1000+).
- **Modo Metralleta**: The Planner instructs the Actor to "Fire all guns", and the Actor MUST output a JSON list with multiple simultaneous tool calls (e.g., `[{"action": "run", "cmd": "ping 1..."}, {"action": "run", "cmd": "ping 2..."}]`).
- **Execution**: The system must verify capacity to spawn N independent processes/threads.
    - *Challenge*: n8n native nodes might bottleneck. May require a custom "Swarm Node" or external worker pool (e.g., Python `multiprocessing`, Docker swarm, or Kubernetes jobs).
    - **Action**: Verify if `Execute Command` node can handle concurrent heavy loads or if we need a dedicated "Batch Executor" via Python script.

### 2. Reduce Phase (The Funnel) 🌪️
- **Aggregation**: Collect stdout/stderr/results from all N instances.
- **Data Volume**: This will produce a massive amount of text context.

### 3. AI Summarization Layer (The Compressor) 🧠
- **Problem**: Passing 1000 results back to the Planner will blow up the context window (Input Token Limit).
- **Solution**: A dedicated "Summarizer Agent" node.
    - **Input**: Raw massive JSON output from Reduce phase.
    - **Goal**: "Summarize what happened, highlight errors, and extract key insights."
    - **Output**: A concise textual summary (e.g., "998 commands succeeded. 2 failed with permission error on /root. Directory size is 45GB.").

## Implementation Steps
1. [ ] **Scalability Test**: Stress test current n8n `Execute Command` with 50 -> 100 -> 500 parallel inputs.
2. [ ] **Executor Logic**: Design a Python script or n8n sub-workflow specifically for high-concurrency dispatch.
3. [ ] **Summarizer Node**: Implement a Gemini 1.5 Flash node solely for compressing the output of these batch jobs.
4. [ ] **Integration**: Connect Actor -> Monster Executor -> Summarizer -> Planner.
