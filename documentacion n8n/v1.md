# 🏁 Hito V1: Vega "Vega" Kernel
**Fecha**: 2026-02-03
**Versión**: 1.0 (Stable Milestone)
**Estado**: Operacional / Hardened

---

## 🏗️ Arquitectura del Sistema
"Vega" es un agente ReAct recursivo implementado sobre n8n, capaz de razonamiento paralelo ("Monster Mode" ready) y ejecución de herramientas nativas del sistema.

### 🔄 Ciclo de Vida (Workflow Loop)
El sistema opera en un bucle continuo de **Percepción -> Razonamiento -> Acción -> Aprendizaje**.

1.  **Planner (Brain)**:
    *   **Modelo**: Gemini 2.0 Flash (vía Custom Node `n8n-nodes-gemini-cli-local`).
    *   **Función**: Recibe el objetivo del usuario (`user_goal`) y el historial (`history`). Decide la siguiente acción.
    *   **Output**: JSON estructurado (Markdown) con `actions: []`.

2.  **Clean Actor (The Firewall)**:
    *   **Tipo**: Code Node (JavaScript).
    *   **Función**: Sanitiza la salida del LLM.
    *   **Logros V1**:
        *   **Markdown Parsing**: Soporta bloques \`\`\`json.
        *   **Drive ID Guard**: Intercepta intentos de usar rutas de archivo (`folder/file`) y fuerza el uso de IDs, previniendo errores 404.
        *   **Robust Input**: Usa `$input.all()[0]` para tolerar cambios en el motor de ejecución de n8n.

3.  **Tool Router**:
    *   **Tipo**: Switch Node.
    *   **Función**: Despacha la acción a nodos especializados (`Drive Search`, `Drive Download`, `Execute Command`, etc.).

4.  **Tools (Los Brazos)**:
    *   *Google Drive*: Búsqueda, descarga, subida (requiere IDs válido).
    *   *System Terminal*: Ejecución de comandos Linux (controlado).
    *   *Read File*: Lectura de contexto local.

5.  **Aggregator (The Funnel)**:
    *   **Tipo**: Code Node (con soporte Parallel).
    *   **Función**: Recolecta resultados de **múltiples** ramas de ejecución (si el Planner disparó varias acciones).
    *   **Mejora V1**: Normaliza salidas dispares (archivos vs stdout) en un string coherente.

6.  **Update State (Memory)**:
    *   **Tipo**: Code Node.
    *   **Función**: Actualiza el historial JSON y el contador de iteraciones.
    *   **Seguridad**: Capa de seguridad (max 25 iteraciones) para evitar bucles infinitos.

---

## 🛡️ Protocolos de Seguridad (Hardening)
*   **Active Defense**: El sistema se niega a ejecutar acciones de Drive destructivas o erróneas si detecta patrones de "ruta de archivo".
*   **Execution Robustness**: Todos los nodos críticos usan la lógica `$input.all()` para garantizar que el estado fluya independientemente de si n8n agrupa los ítems o los separa.

## 🚀 Monster Mode (Ready)
El sistema está preparado arquitectónicamente para paralelismo.
*   Si el Planner emite 5 acciones (`ls`, `search`, `cat`...), el `Clean Actor` genera 5 items.
*   n8n los procesa en paralelo (o secuencia rápida).
*   El `Aggregator` los recompone en un solo reporte de estado para el siguiente ciclo del Planner.
